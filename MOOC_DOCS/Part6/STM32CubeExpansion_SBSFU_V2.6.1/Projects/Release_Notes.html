Release Notes for Projects of X-CUBE-SBSFU Expansion package

Copyright © 2017 STMicroelectronics

Purpose

The package includes different sample applications to provide a complete SBSFU solution:

  • SECoreBin application: provides a binary including all the “trusted” code

  • SBSFU application:
    – Secure Boot (Root of Trust)
    – Local download via UART Virtual COM or over BLE for STM32WB Series
    – FW installation management

  • User application:
    – Downloads a new firmware in dual slot configuration
    – Provides examples testing protection mechanisms

In the X-CUBE-SBSFU Expansion package, the projects are organized in a sequential scheme based on the following levels:

  • Boards : Hardware platform
  • Project Category: Applications
  • Project Class
  • Project Subclass

The Project Class refers to the configuration of the SBSFU solution :

  • Dual slot configuration: 2_Images folder
  • Dual slot on external flash configuration: 2_Images_ExtFlash folder
  • Single slot configuration: 1_Image folder
  • mbed TLS cryptographic services (Open Source Crypto) configuration (2_Images_OSC / 1_Image_OSC folders)
  • STSAFE-A configuration: 2_Images_STSAFE folder
  • KMS configuration : 2_Images_KMS folder

The Project Subclass refers to one of the projects part of the complete SBSFU solution (SECoreBin, SBSFU, UserApp, Loader, KMS_Blob).

X-CUBE-SBSFU is provided as reference code to demonstrate state-of-the-art usage of the STM32 security protection mechanisms. It is a starting point for OEMs to develop their own SBSFU as a function of their product security requirement levels.

Details on the available sample applications per board is available in SBSFU Projects List

Main Changes

  • Add the support of STM32H7B3I-DK and NUCLEO-G031K8 boards.

  • Specific SBSFU application introduced for STM32H7B3I-DK Discovery Kit supporting external flash and OTFDEC: active image is stored in the external memory and remains encrypted to ensure confidentiality of the firmware. On the fly decryption is performed by OTFDEC IP during user application execution from external flash.

  • Double-bit ECC improvements : Changes in management of exceptions for all STM32 series supporting Flash ECC error (applicable to STM32L4, STM32G0, STM32G4, STM32H7 and STM32WB series): fix on double ecc error management (problem could be reproduced in erase/write operation in Flash, during a power off or a reset)

  • Slot0 Header not contiguous with FW image feature added and deployed in STM32H7B3I-DK External flash variant

  • Complete header is now integrated into UserApp.sfb file. No more padding required during the firmware image download

  • Change SRAM define (SRAM1) to comply with STM32 diversity (SRAM1, SRAM2, SRAM3)

  • Add 1_image variant for P-NUCLEO-WB55.Nucleo board: this new example is based on standalone loader, to be easily replaced by BLE loader. Decrypt in place function added (DecryptImageInSlot0 ) and installation process updated with a new decrypt in place function

  • Add NUCLEO-G031K8 board: debug prompts are deactivated, and example is restricted to AES cryptographic scheme

  • DMAMUX registers are now restricted to Privileged software on STM32G0 series

  • Time constant comparisons added in SE_CoreBin

  • Update of Pre-Build and Post build scripts

  • Update MPU configuration issue of User Application with 2 Images on P-NUCLEO-WB55.Nucleo board

Add the support of STM32L4 combined with STSAFE-A100 boards.

Add support of mbed TLS cryptographic services (Open Source Crypto) for STM32F413H-Discovery and  STM32F769I-Discovery boards.

Integration of STSAFE-A middleware. Example provided on the B-L475E-IOT01A board.

Integration of KMS middleware. Example provided on the B-L475E-IOT01A board.

Add a standalone Loader project implementing the firmware download feature (over Ymodem protocol) in order to make easier the integration of over the air download capability (BLE protocol for STM32WB Series). Example provided on the P-NUCLEO-WB55.Nucleo board.

Known Limitations

Microsoft Windows has a limitation whereby paths to files and directories can’t be longer than 256characters. Paths to files exceeding that limits cause tools (e.g. compilers, shell scripts) to fail reading from or writing to such files. As a workaround, it’s advised to use the subst.exe command from within a command prompt to set up a local drive out of an existing directory on the hard drive, such as:

C:\> subst X:<PATH_TO_CUBEFW>\STM32CubeExpansion_SBSFU_Vx.y.z
