Purpose

The X-CUBE-SBSFU Secure Boot and Secure Firmware Update solution allows the update of the STM32 microcontroller built-in program with new firmware versions, adding new features and correcting potential issues. The update process is performed in a secure way to prevent unauthorized updates and access to confidential on-device data.

The Secure Boot (Root of Trust services) is an immutable code, always executed after a system reset, that checks STM32 static protections, activates STM32 runtime protections and then verifies the authenticity and integrity of user application code before every execution in order to ensure that invalid or malicious code cannot be run.

The Secure Firmware Update application receives the firmware image via a UART interface with the Ymodem protocol, checks its authenticity, and checks the integrity of the code before installing it. The firmware update is done on the complete firmware image, or only on a portion of the firmware image.

Examples are provided for single slot configuration configuration in order to maximize firmware image size, and for dual slot configuration in order to ensure safe image installation and enable over-the-air firmware update capability commonly used in IOT devices.

Examples can be configured to use asymmetric or symmetric cryptographic schemes with or without firmware encryption.

The secure key management services provide cryptographic services to the user application through the PKCS # APIs (KEY ID based APIs) that are executed inside a protected and isolated environment. User application keys are stored in the protected and isolated environment for their secured update: authenticity check, data decryption and data integrity check. This is available on the STM32L4 Series with example provided on the B-L475E-IOT01A and B-L4S5I-IOT01 boards.

STSAFE-A110 is a tamper-resistant secure element (HW Common Criteria EAL5+ certified) used to host X509 certificates and keys, and perform verifications that are used for firmware image authentication during Secure Boot and Secure Firmware Update procedures.This is available on the STM32L4 Series with example provided on the B-L4S5I-IOT01 board.

X-CUBE-SBSFU is built on top of STM32Cube software technology, making the portability across different STM32 microcontrollers easy. It is provided as reference code to demonstrate best use of STM32 security protections.

The X-CUBE-SBSFU Expansion Package comes with examples running on the STM32F4 Series, STM32F7 Series, STM32G0 Series, STM32G4 Series, STM32H7 Series, STM32L0 Series, STM32L1 Series, STM32L4 Series, and STM32WB Series. An example combining STM32 microcontroller and STSAFE-A110 is also provided for the STM32L4 Series. The figure below shows the overall architecture. 

Note: For STM32L5 Series, SBSFU solution is based on the Open Source TF-M reference implementation and delivered in STM32CubeL5 package ; for STM32WL Series, SBSFU for Dual Core is delivered in STM32CubeWL package.

  • Microsoft Windows has a limitation whereby paths to files and directories can’t be longer than 256 characters. Paths to files exceeding that limits cause tools (e.g. compilers, shell scripts) to fail reading from or writing to such files. Main Changes

  • New board B-L4S5I-IOT01A introduced in this package. This board integrates :
    • STM32L4S5I MCU (Cortex(R)-M4 at 120 MHz, 2 Mbytes of Flash memory)
    • secure element STSAFE-A110

  • New application is delivered on B-L475E-IOT01A to demonstrate how to configure slot of download in external flash

  • New features introduced in KMS : Object search, ECC key pair generation, ECDH key derivation, Lock keys and services vendor APIs, and improvement of memory management (introduction of different allocators)

  • Management of interruption during code execution inside the firewall is now supported for applications requiring low latency on interruption handling. IRQ are no more disabled when entering into the isolated code environment, when IT_MANAGEMENT compilation switch is enable.

  • The firmware image configuration can be extended up to 3 images for complex system with multiple firmwares:
    • up to 3 active slots and 3 download slots for complex system with multiple firmwares
    • specific cryptographic keys per slot,
    • simultaneous image installation in order to ensure compatibility between firmwares,
    • possibility of installation with SWAP area in order to limit memory overhead needed. Rollback option is then possible.
  • Add image state handling feature: user application should validate the installation of new active image(s) in case of successfull validation of new image through “self-test”.
    • The active image identified as MASTER will validate all new installed active images in a single operation.
    • In case of failure in process of validation of new image(s), active images can be rolled back to their previous version (if identified, else erased)

  • Update prepare image tools to support new functionalities described above (multi-images, header is not contiguous with firmware image, image state handling)

Main Changes

Add the support of NUCLEO-L073RZ, NUCLEO-G474RE, NUCLEO-H753ZI, NUCLEO-L152RE, P-NUCLEO-WB55.Nucleo boards.

Add the support of STM32L4 combined with STSAFE-A100 boards.

Add the partial update capability on all platforms. A partial image contains only the binary portion of the new firmware image to install, versus the active firmware image. Partial image usage presents various benefits:
- smaller firmware image to download (reduced download duration)
- faster firmware image installation
- memory mapping optimization, with possibly smaller slot #1 (maximum size of partial image) and bigger slot #0 (maximum size of full image)

This feature is available on dual firmware images variant only (not available on single firmware image variant).

Integration of KMS middleware : The secure key management services provide cryptographic services to the user application through the PKCS # APIs (KEY ID based APIs) that are executed inside a protected and isolated environment. User application keys are stored in the protected and isolated environment for their secured update: authenticity check, data decryption and data integrity check. This is available on the STM32L4 Series with example provided on the B-L475E-IOT01A board.

Integration of STSAFE-A middleware : STSAFE-A100 is a tamper-resistant secure element (HW Common Criteria EAL5+ certified) used to host X509 certificates and keys, and perform verifications that are used for firmware image authentication during Secure Boot and Secure Firmware Update procedures. This is available on the STM32L4 Series with example provided on the B-L475E-IOT01A board.

Add a standalone Loader project implementing the firmware download feature (over Ymodem protocol) in order to make easier the integration of over the air download capability (BLE protocol for STM32WB Series). Example provided on the P-NUCLEO-WB55.Nucleo board.

Integration of STM32 WPAN middleware to setup mailbox communication between M4 and M0+ for CKS (Customer Key Services) feature on P-NUCLEO-WB55.Nucleo. SBSFU symmetric firmware encryption key is hidden into M0+ core side.

General improvements (software counter measures) against software attacks and against basic hardware faults injection.

Firmware installation resume implemented (replacing automatic rollback in case of installation failure).
Automatic rollback after 3 consecutive errors removed.

Add support of mbed TLS cryptographic services (Open Source Crypto) for STM32F413H-Discovery and  STM32F769I-Discovery boards. New projects created : 2_Images_OSC.
