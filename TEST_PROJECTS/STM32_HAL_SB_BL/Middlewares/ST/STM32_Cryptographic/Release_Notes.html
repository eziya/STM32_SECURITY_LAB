Release Notes for STM32 Cryptographic middleware

Copyright © 2005 STMicroelectronics

Purpose

  • STM32 Cryptographic middleware is composed of libraries that includes a large set of cryptographic algorithms based on firmware implementation ready to use in all STM32 Series. The main algorithms are :

    • Cipher algorithms : AES (modes CBC, CCM, CFB, CTR, ECB, GCM, OFB, XTS), SM4 (modes CBC, CFB, CTR, ECB, OFB), Chahcha20

    • Hash methods : SHA-1, SHA-2, SHA-3, MD5, SM3, SHAKE

    • Message Authentication Code : HMAC, AES MAC, KMAC

    • Asymmetric : ECDSA, EdDSA, SM2, ECDH, RSA v2.2

    • Random generator

  • Cryptographic Firmware Library package based on STM32Cube architecture.

  • Library optimized for STM32 Series based on Cortex M0/M0+, Cortex-M3, Cortex-M4, Cortex-M7, Cortex-M33.

  • EABI : Cryptographic libraries are compatible with GCC, ARM and EWARM compilers.

  • ACVP : Cryptographic algorithms and their individual components are validated by the NIST, for each Cortex-M architecture, and for fastest and smallest implementations.
    URL of validation reports stored on NIST portal are described in “Released Libraries” table
